# 站内信与公告改版 PRD

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 站内信与公告改版 PRD |
| 版本 | V1.0 |
| 状态 | 需求确认稿 |
| 适用后台 | 运营后台 / 用户触达 |
| 原型 | `outputs/user-reach-config.html` |

## 2. 背景与问题

原后台将通知、站内信、公告拆成三个入口，但三者在系统事件、触达渠道、内容配置、APP 推送和统计结果上存在重复：

1. 通知中的站内信触达与站内信配置重复，全站公告触达与公告配置重复。
2. 站内信列表同时承担配置管理和用户触达结果查看，统计对象不清晰。
3. 定时触达能力只支持简单时间，无法覆盖每天、每周、每月及指定时间的多次推送。
4. 触达渠道、内容、跳转、礼金、弹窗之间缺少完整联动，容易产生无效或矛盾配置。
5. 字段命名、列表字段、筛选字段和汇总指标缺少统一口径。

本次改版取消独立“通知”菜单，将通知能力合并到站内信和公告，并拆分配置管理与用户维度统计。

## 3. 改版目标

1. 建立“配置规则”和“触达结果”两类清晰对象。
2. 使用站内信配置、公告配置统一承接手动、定时和系统事件触达。
3. 统一站内信、公告与 APP 推送的渠道配置、多语言内容和统计口径。
4. 支持完整的人群圈选、周期推送、跳转、礼金和弹窗联动。
5. 保证配置可复制、可启停、可追溯，统计可筛选、可汇总、可导出。

## 4. 范围

### 4.1 本期范围

- 删除“通知”菜单。
- 新增并改版四个二级菜单：站内信统计、站内信配置、公告统计、公告配置。
- 将原通知事件迁移为站内信系统推送或系统公告。
- 支持 APP 推送附加渠道及其多语言内容配置。
- 支持用户维度明细、筛选联动汇总、详情和导出。
- 支持手动、定时、系统事件三类配置。

### 4.2 非本期范围

- 不新增独立通知配置或通知统计入口。
- 不提供配置删除；以启用、停用管理生命周期。
- 不建设单条站内信或单条公告维度的独立统计页面。
- 不在配置列表展示事件类型、跳转目标等下级或低频配置字段。
- 不在条件用户组件中提供包ID或渠道ID条件。

## 5. 信息架构

| 菜单 | 对象维度 | 核心职责 |
|---|---|---|
| 站内信统计 | 用户触达记录 | 查看谁收到了哪条站内信及阅读、点击、礼金、APP 触达结果 |
| 站内信配置 | 规则配置 | 维护手动、定时、系统推送规则及站内信内容 |
| 公告统计 | 用户触达记录 | 查看谁收到了哪条公告及阅读、点击、弹窗、APP 触达结果 |
| 公告配置 | 规则配置 | 维护手动、定时、系统公告规则、公告内容和弹窗配置 |

| 充值失败 | 站内信配置 / 公告配置 | 系统推送 / 系统公告 | 充值失败 |
| 提现成功 | 站内信配置 / 公告配置 | 系统推送 / 系统公告 | 提现成功 |
| 提现失败 | 站内信配置 / 公告配置 | 系统推送 / 系统公告 | 提现失败 |
| 奖励未领取 | 站内信配置 | 系统推送 | 奖励未领取 |
| 新人注册当日充值提醒 | 站内信配置 | 系统推送 | 新人注册当日充值提醒 |
| 新人未充值每日提醒 | 站内信配置 | 系统推送 | 新人未充值每日提醒 |
| 新手任务每日提醒 | 站内信配置 | 系统推送 | 新手任务每日提醒 |
| 梦幻体育每日推送 | 站内信配置 / 公告配置 | 定时推送或系统事件 | 梦幻体育每日推送 |
| 梦幻体育比赛开始前提醒 | 站内信配置 / 公告配置 | 系统推送 / 系统公告 | 梦幻体育比赛开始前提醒 |

## 7. 通用业务规则

### 7.1 配置生命周期

- 新增配置保存成功后默认状态为“未启用”，不设置草稿状态。
- 状态枚举：未启用、已启用、已停用。
- 未启用或已停用配置可启用；已启用配置可停用。
- 启用、停用均需二次确认。
- 配置不可删除。
- 编辑修改原配置；复制创建新配置，不修改原配置。
- 复制需回填全部可编辑内容，包括包ID、推送规则、条件、开关、可增删行、全部语言内容和素材引用；不复制系统ID、状态、修改人和修改时间。

### 7.2 包ID

- 包ID用于表达配置归属，支持多选，首项固定为 `all`。
- `all` 与具体包ID互斥。
- 新增时默认零项选中，显示“请选择”。
- 配置为指定用户时，包ID清空、置灰并显示“无需配置”，保存时不校验包ID。
- 切换回全体用户或条件用户后恢复可配置和必填，不恢复旧值。
- 统计中的用户来源独立于包ID，枚举统一为 `Ads`、`Invite`、`H5`。

### 7.3 推送类型与推送范围

| 推送类型 | 默认推送范围 | 下级配置 |
|---|---|---|
| 手动推送 / 手动公告 | 指定用户 | 用户ID列表 |
| 定时推送 / 定时公告 | 全体用户 | 推送周期与多条推送时间 |
| 系统推送 / 系统公告 | 条件用户，且不可切换 | 条件类型固定包含事件类型，事件值必选 |

切换推送类型时，需重置不再适用的推送范围、时间和事件配置。系统类型的事件类型属于下级配置，只出现在抽屉，不进入配置列表或筛选区。

### 7.4 定时推送

- 推送周期为必填下拉，枚举：每天、每周、每月、自定义。
- 每天：配置一个或多个推送时间。
- 每周：每行配置星期与推送时间，可配置多行，例如周三 12:00、周三 13:00、周四 11:00。
- 每月：每行配置日期与推送时间，可配置多行，例如 1 号 12:00、1 号 13:00。
- 自定义：每行配置指定日期与推送时间。
- 新增入口位于组件右上角；每行右侧删除当前行；至少保留一行。
- 切换周期时清空并重建时间行，不保留旧周期数据。

### 7.5 系统事件特殊参数

- 新人注册当日充值提醒：增加“注册后未充值提醒时间”，输入大于 0 的整数，单位分钟。
- 梦幻体育比赛开始前提醒：增加“比赛开始前提醒时间”，输入大于 0 的整数，单位分钟。
- 特殊参数归入规则配置，只在对应系统事件下显示；切换事件或推送类型后隐藏并清空。

### 7.6 多语言

- 当前语言：英语、印地语。
- 每种语言独立保存标题、正文、按钮文案和 APP 推送内容。
- 切换语言只能切换当前语言内容，不得改变礼金、APP 推送、范围、跳转或定时配置状态。
- 名称上限 50 字；站内信标题、公告标题、按钮文案、APP 推送标题和 APP 推送内容上限 36 字，计数显示在输入框内部右侧。

## 8. 站内信配置

### 8.1 列表与筛选

筛选字段顺序：站内信名称、站内信ID、包ID、推送类型、推送范围、是否开启APP推送、是否开启礼金配置、修改人、修改时间、配置状态、查询、重置。

列表字段顺序：序号、站内信名称、站内信ID、包ID、推送类型、推送范围、预计推送人数、是否开启APP推送、是否开启礼金配置、修改人、修改时间、配置状态、操作。

操作：编辑、复制、详情、启用/停用。列表不提供删除。

### 8.2 新增与编辑抽屉

#### 基本信息

| 字段 | 必填 | 控件与规则 |
|---|---|---|
| 站内信名称 | 是 | 输入框，最多 50 字 |
| 包ID | 条件必填 | 下拉多选；指定用户时不适用 |
| 推送类型 | 是 | 手动推送、定时推送、系统推送 |
| 定时推送时间配置 | 条件必填 | 仅定时推送显示 |

站内信ID由系统保存后生成，不在新增/编辑抽屉展示。

#### 推送配置

- 触达方式使用复选组。
- “站内信”默认勾选且不可取消。
- “APP推送”为可选附加渠道。

#### 参与条件

- 推送范围：全体用户、指定用户、条件用户。
- 指定用户：输入一个或多个用户ID。
- 条件用户：使用可增删条件组件，条件类型驱动条件值。
- 系统推送固定为条件用户，并默认生成事件类型条件。
- 展示预计推送人数和覆盖率。

#### 规则配置

1. 基础规则配置
   - 是否开启站内信按钮跳转。
   - 开启后，跳转类型、跳转目标以及每种语言的站内信按钮文案必填。
   - 跳转类型：功能页、Web页、外链；开启跳转后不提供“不跳转”。
   - 关闭后隐藏并清空跳转类型、目标和所有语言按钮文案。
2. 礼金配置
   - 是否开启礼金配置。
   - Tooltip：默认包含在奖励列表中展示；推送范围为全体用户时不可开启礼金配置。
   - 开启后配置礼金金额、礼金倍率、礼金适用范围、适用范围值。
   - 礼金适用范围为无限制时不显示范围值；指定场馆、游戏分类或游戏时显示多选范围值。
   - 是否开启弹窗领取位于礼金配置第一行；开启后用户可通过弹窗领取。
   - 推送范围切换为全体用户时，强制关闭并禁用礼金配置，清空礼金参数和弹窗领取状态。

#### 前台展示

1. 站内信配置
   - 每种语言配置站内信标题和站内信内容。
   - 站内信内容使用富文本编辑器。
   - 开启站内信按钮跳转时，站内信按钮文案显示在站内信标题后并必填。
2. APP推送配置
   - 仅勾选 APP推送时显示。
   - 每种语言配置 APP推送标题、APP推送内容，均必填且最多 36 字。
   - 取消勾选 APP推送后隐藏配置、停止校验并清空所有语言的 APP 内容；重新勾选保持空值。

## 9. 站内信统计

### 9.1 统计对象

按用户触达记录展示，一行代表某用户收到某条站内信的结果。支持按站内信名称和时间查询，不新增单条站内信统计页面。

### 9.2 筛选

用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源、站内信名称、站内信ID、推送类型、触达方式、已读状态、是否赠送礼金、领取状态、发送时间、发送状态。

操作顺序固定为：查询、重置、导出。导出为文字按钮，导出当前筛选结果。

### 9.3 汇总概览

| 指标 | 口径 |
|---|---|
| APP推送人数 | APP 推送成功的用户ID去重数 |
| APP推送开启人数 | 已开启 APP 通知权限的用户ID去重数 |
| APP推送点击人数 | 点击 APP 推送入口的用户ID去重数 |
| APP推送点击率 | APP推送点击人数 / APP推送人数，分母为 0 显示 0% |
| 站内信推送人数 | 成功推送站内信的用户ID去重数 |
| 站内信阅读人数 | 已阅读站内信的用户ID去重数 |
| 站内信按钮点击人数 | 点击站内信按钮的用户ID去重数 |
| 站内信转化率 | 阅读用户与按钮点击用户并集去重数 / 站内信推送人数 |
| 礼金发放额 | 当前结果中已赠送礼金金额合计 |
| 礼金领取额 | 当前结果中领取状态为已领取的礼金金额合计 |

汇总指标随查询和重置同步刷新，使用一条紧凑横向指标带展示。

### 9.4 明细与详情

列表优先展示：序号、用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源、站内信名称、站内信ID、推送类型、触达方式、已读状态、是否赠送礼金、礼金金额、领取状态、发送时间、发送状态、操作。

详情分组：

- 用户信息：用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源。
- 站内信信息：站内信名称、站内信ID、跳转类型、推送类型、推送范围、触达方式、是否赠送礼金、礼金金额、发送时间。
- 触达结果：已读状态、读取时间、站内信点击状态、领取状态、领取时间、APP推送状态、通知权限状态、APP推送点击状态。

详情不展示失败原因和重复的发送状态。

## 10. 公告配置

### 10.2 新增与编辑抽屉

#### 基本信息

## 11. 公告统计

### 11.1 统计对象

按用户触达记录展示，一行代表某用户收到或看到某条公告的结果。

### 11.2 筛选

用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源、公告名称、公告ID、推送类型、是否弹窗展示、阅读状态、点击状态、展示时间、状态。

操作顺序固定为：查询、重置、导出。导出当前筛选结果。

### 11.3 汇总概览

沿用站内信统计的渠道指标结构：APP推送人数、APP推送开启人数、APP推送点击人数、APP推送点击率、公告推送人数、公告阅读人数、公告按钮点击人数、公告转化率。

公告转化率 = 阅读用户与按钮点击用户并集去重数 / 公告推送人数；分母为 0 显示 0%。公告无礼金业务时不展示礼金指标。

### 11.4 明细与详情

列表优先展示：序号、用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源、公告名称、公告ID、推送类型、是否弹窗展示、展示时间、阅读状态、点击状态、状态、操作。

详情分组：

- 用户信息：用户ID、用户昵称、VIP等级、账号类型、包ID、用户来源。
- 公告信息：公告名称、公告ID、跳转类型、推送类型、推送范围、触达方式、是否弹窗展示、弹窗展示位置、展示时间。
- 触达结果：阅读状态、阅读时间、点击状态、APP推送状态、通知权限状态、APP推送点击状态。

## 12. 字段与交互约束

- 多枚举单选使用下拉框；只有可复选场景才平铺复选项或使用下拉多选。
- 独占一行的普通控件保持标准宽度，不拉伸填满整行。
- 所有联动下级字段默认“请选择”或空值，出现后标记必填并参与保存校验。
- 修改上级字段时递归清空全部后续层级。
- 输入长度计数显示在输入框内部右侧。
- 字段说明优先放在字段名后的问号 Tooltip 中，不单独占行。
- Tooltip 不得被抽屉、表格或滚动容器裁切。
- 表格列宽按标题和内容自适应，不均分；保留竖向分割线。
- 筛选字段与表格同名字段保持同一标准名、枚举口径和相对顺序。

## 13. 权限与审计

- 查询权限：查看统计列表、汇总和详情。
- 导出权限：导出当前筛选结果。
- 配置查看权限：查看配置列表和详情。
- 配置编辑权限：新增、编辑、复制。
- 配置状态权限：启用、停用。
- 保存配置时记录创建人、创建时间、修改人、修改时间。
- 系统事件自动执行记录系统标识，人工修改记录实际操作人。

## 14. 数据与接口要求

1. 配置接口需区分站内信和公告对象，并支持手动、定时、系统事件三类规则。
2. 定时任务需支持周期、多行时间和时区，避免重复执行。
3. 系统事件需保证幂等，同一业务事件和用户不得重复生成相同触达记录。
4. 统计明细以用户ID + 内容ID + 触达批次作为可追踪主线。
5. APP 推送、站内信、公告结果需独立记录，支持统一汇总。
6. 阅读、点击和领取时间记录首次有效行为时间；重复行为不重复计人数。
7. 导出使用与当前筛选相同的数据集合和字段口径。

## 15. 前端页面联动

### 15.1 通用命中与展示链路

前端请求站内信、公告或弹窗数据时，服务端按以下顺序判断：

1. 配置状态为已启用。
2. 当前时间命中配置生效周期及推送时间。
3. 当前用户所属包ID命中配置包ID；配置为 `all` 时命中全部包。
4. 当前用户命中推送范围、指定用户或条件用户规则。
5. 当前用户语言命中对应语言内容。
6. 同一配置、用户和触达批次未被重复生成或重复展示。

任一条件不满足时，前端不展示该内容。后台停用配置后不再产生新的触达；已经生成的站内信记录仍保留在用户站内信列表中，历史统计记录不删除。公告停用或超过生效周期后不再出现在公告展示入口和弹窗队列中。

### 15.2 站内信前端联动

#### 站内信列表

- 用户命中配置后生成一条用户维度站内信记录，并进入站内信列表。
- 列表至少展示站内信标题、摘要、发送时间、已读/未读状态。
- 新站内信增加未读数量；同一触达记录不得重复增加未读数。
- 列表内容使用用户当前语言对应的站内信配置。
- 用户点击列表项进入站内信详情，并回传首次读取时间；重复打开不重复计算阅读人数。

#### 站内信详情

- 展示站内信标题和富文本内容。
- “是否开启站内信按钮跳转”为关闭时，不展示按钮。
- 开启时展示当前语言的站内信按钮文案；点击后按跳转类型和跳转目标执行，并回传站内信按钮点击事件。
- 跳转目标不可用时保留当前详情页并给出前端失败反馈，不得进入空白页。

#### 站内信礼金

- 开启礼金配置后，礼金默认进入用户奖励列表，无需额外配置奖励列表入口。
- 奖励列表展示礼金金额、礼金倍率、适用范围、领取状态和有效状态。
- 开启弹窗领取后，用户命中站内信时可出现礼金领取弹窗；未开启时只能从奖励列表领取。
- 用户在弹窗或奖励列表任一入口领取后，两个入口的领取状态必须实时同步，禁止重复领取。
- 领取成功记录领取时间；过期、已领取或不满足范围时前端不可重复领取。

### 15.3 公告前端联动

#### 公告列表与详情

- 命中的公告进入前端公告列表或公告入口。
- 公告列表至少展示公告标题、摘要和展示时间。
- 用户打开公告详情时回传首次阅读时间；重复打开不重复计算阅读人数。
- 公告详情展示当前语言的公告标题、富文本内容和可选跳转按钮文本。
- 配置为不跳转时不执行页面跳转；配置其他跳转类型时按目标打开对应页面，并回传公告按钮点击事件。

#### 首页弹窗

- “弹窗是否展示”为关闭时，公告只通过公告入口展示，不进入弹窗队列。
- 开启后，弹窗默认在首页展示，不提供其他展示位置配置。
- 多个弹窗同时命中时，按弹窗展示顺序从大到小排列；顺序相同按生效时间或配置更新时间稳定排序。
- 同一用户再次展示同一弹窗需满足弹窗间隔时间。
- 弹窗展示图片取已上传的单图素材；素材不可用时不得展示破损图片，可降级为纯内容弹窗或跳过该弹窗并上报异常。
- 弹窗按钮按弹窗跳转类型和目标执行；无跳转时只提供关闭操作。
- 前端需回传弹窗曝光、关闭和按钮点击事件。

### 15.4 APP 推送联动

- 未勾选 APP推送时，不生成 APP 推送任务，前端也不展示 APP 通知。
- 勾选后，使用用户当前语言对应的 APP推送标题和 APP推送内容生成系统通知。
- 用户未开启通知权限时记录通知权限状态，但不影响默认站内信或公告渠道生成内容。
- APP 推送点击默认打开对应的站内信详情或公告详情；内容内部跳转仍由站内信/公告按钮配置控制。
- APP 推送送达和点击结果需分别回传，供 APP推送人数、APP推送点击人数和APP推送点击率统计。
- 同一触达批次内，APP 推送与站内信/公告共用内容ID和用户ID关联，便于跨渠道追踪，但各渠道结果独立记录。

### 15.5 多语言联动

- 前端根据用户当前语言读取同一配置下的对应语言内容。
- 站内信标题、正文、按钮文案、公告标题、公告正文、APP推送标题和内容均按语言独立读取。
- 用户在内容生成后切换语言时，历史站内信展示口径由产品实现选择“发送快照”或“实时语言内容”，但同一项目必须保持一致；本期建议保存发送时语言快照，避免后台修改影响历史消息。
- 后台已对所有启用渠道的语言字段做必填校验，前端不应收到缺少标题或正文的有效配置。

### 15.6 跳转联动

| 跳转类型 | 前端行为 |
|---|---|
| 不跳转 | 不展示跳转目标，不执行跳转 |
| 功能页 | 使用前端路由打开配置的功能页 |
| Web页 | 在产品约定的 WebView 或站内网页容器打开目标页 |
| 外链 | 校验合法 URL 后通过安全外链能力打开 |

- 跳转前需校验目标存在、用户有访问权限且目标在当前包可用。
- 跳转成功和失败均需回传；统计中的按钮点击以用户完成有效点击为准，不以目标页面加载成功作为点击计数前提。
- 后台修改跳转类型时会清空旧目标，前端不得缓存并继续使用旧目标。

### 15.7 前端状态与埋点

| 事件 | 触发时机 | 核心参数 |
|---|---|---|
| 站内信发送 | 生成用户站内信记录 | 用户ID、站内信ID、触达批次、包ID、发送时间 |
| 站内信阅读 | 首次打开站内信详情 | 用户ID、站内信ID、读取时间 |
| 站内信按钮点击 | 点击站内信按钮 | 用户ID、站内信ID、跳转类型、跳转目标、点击时间 |
| 公告展示 | 公告在列表、详情或弹窗有效曝光 | 用户ID、公告ID、展示入口、展示时间 |
| 公告阅读 | 首次打开公告详情 | 用户ID、公告ID、阅读时间 |
| 公告按钮点击 | 点击公告或弹窗按钮 | 用户ID、公告ID、展示入口、跳转类型、点击时间 |
| 弹窗关闭 | 用户主动关闭公告弹窗 | 用户ID、公告ID、关闭时间 |
| 礼金领取 | 礼金领取成功 | 用户ID、站内信ID、领取入口、礼金金额、领取时间 |
| APP推送送达 | 推送服务返回成功 | 用户ID、内容ID、触达批次、送达时间 |
| APP推送点击 | 用户点击系统通知 | 用户ID、内容ID、触达批次、点击时间 |

所有事件需包含唯一事件ID并支持幂等去重。阅读、点击、领取等人数指标按用户ID去重，但原始行为记录允许保留多次行为用于排查。

### 15.8 前端异常处理

- 配置已停用、已失效或用户不再命中时，前端刷新后移除尚未展示的公告和弹窗任务。
- 跳转目标失效、图片加载失败、富文本解析失败、APP 推送服务失败时需记录错误码和内容ID。
- 富文本需过滤不安全标签和脚本，外链需经过白名单或安全校验。
- 网络失败时，阅读、点击、关闭和领取事件需支持本地重试；领取操作必须以服务端最终结果为准。

## 16. 验收标准

### 16.1 菜单与迁移

- 后台不再显示通知菜单。
- 原通知能力可在站内信配置或公告配置中完成等价配置。
- 历史配置迁移后状态、事件和关键参数正确。

### 16.2 配置

- 手动、定时、系统类型分别显示正确的下级字段和默认范围。
- 定时配置支持每天、每周、每月、自定义及多条时间，且至少保留一条。
- 系统类型固定条件用户并要求选择事件类型。
- 指定用户时包ID正确置灰并免校验。
- 全体用户时礼金配置不可开启。
- 默认触达渠道不可取消；APP 推送可独立勾选。
- 勾选 APP 推送后，各语言 APP 推送标题和内容显示且必填；取消后全部清空。
- 多语言切换不改变其他组件状态。
- 复制后全部可编辑配置完整回填，保存生成新ID且状态为未启用。

### 16.3 统计

- 统计列表按用户维度展示，不混入配置规则维护。
- 查询和重置同步刷新汇总及明细。
- 导出结果与当前筛选结果一致。
- APP、站内信、公告人数均按用户ID去重。
- 点击率和转化率计算符合本 PRD 公式。
- 礼金发放额与领取额随筛选正确变化。
- 详情只展示补充信息，不出现长期空白的失败原因或重复终态字段。

### 16.4 前端联动

- 命中包ID、时间、状态和参与条件的用户才能看到对应站内信、公告或弹窗。
- 站内信阅读、公告阅读、按钮点击、APP 推送点击和礼金领取均能正确回传并驱动统计。
- 站内信按钮开关关闭时前端不显示按钮；开启时显示正确语言文案并按目标跳转。
- 公告弹窗仅在开启状态、有效周期和间隔规则满足时于首页展示，展示顺序正确。
- APP 推送关闭时不产生推送任务；开启时使用对应语言标题和内容，点击进入对应内容详情。
- 礼金在奖励列表默认可见；开启弹窗领取时两种领取入口状态一致且不可重复领取。
- 多语言切换不串值，历史内容按发送快照稳定展示。
- 前端重复上报不造成阅读、点击、领取或统计人数重复累计。

### 16.5 视觉与可用性

- 抽屉按基本信息、参与条件、规则配置、前台展示组织，无内容不强行增加分组。
- 字段、按钮、提示、Tooltip、表格内容无裁切、遮挡或错位。
- 可增删组件的新增入口位于右上角，删除入口位于行尾。
- 桌面端和窄屏下字段、汇总指标和底部操作均可正常使用。

## 17. 上线建议

1. 上线前执行历史通知配置映射和缺失字段检查。
2. 新旧任务并行观察一个触达周期，核对发送人数、点击、阅读和礼金数据。
3. 确认新任务稳定后停用旧通知调度，避免重复触达。
4. 对系统事件、定时任务、APP 推送和礼金发放设置监控与告警。
5. 上线后一周复核统计口径和导出数据，重点检查跨渠道去重与转化率。
